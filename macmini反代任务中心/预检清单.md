# 预检清单

## 1. 使用方式

正式部署前，按这份清单逐项确认。  
任何一项不清楚，都不要直接上线。

## 2. Docker 环境检查

### 2.1 容器基线

- [ ] 已列出现有所有 Docker 容器
- [ ] 已确认哪些容器属于正在运行的重要任务
- [ ] 已确认哪些容器绝对不能重启

### 2.2 Compose 基线

- [ ] 已列出现有所有 Compose 项目
- [ ] 已确认未来新项目不会和现有 Compose 项目重名

### 2.3 网络与卷

- [ ] 已列出现有 Docker 网络
- [ ] 已列出现有 Docker 卷
- [ ] 已确认新项目会使用独立网络名
- [ ] 已确认新项目会使用独立卷名前缀

## 3. 端口检查

- [ ] 已列出当前机器重要端口占用
- [ ] 已确认 `18080` 未被占用
- [ ] 已确认 `16379` 未被占用
- [ ] 已确认 `19090` 未被占用或已另选端口
- [ ] 已确认不会把 Redis 暴露到公网

## 4. 资源余量检查

- [ ] 已确认 CPU 有足够余量支撑 4 个槽位
- [ ] 已确认内存足够支撑 Redis、API 和 worker
- [ ] 已确认磁盘足够存储任务日志与结果目录
- [ ] 已确认现有 Docker 任务本身没有资源告急

## 5. 代理与 VPN 检查

### 5.1 代理来源

- [ ] 已确认 Codex 当前走的是 VPN、系统代理还是环境变量代理
- [ ] 已确认代理变量是否写在 shell profile 中
- [ ] 已确认 Docker 是否会继承这些代理变量

### 5.2 隔离要求

- [ ] 已确认 `cloudflared` 默认不继承 Codex 代理
- [ ] 已确认 API 默认不继承 Codex 代理
- [ ] 已确认 Redis 默认不继承 Codex 代理
- [ ] 已确认普通 worker 默认不继承 Codex 代理
- [ ] 已确认仅 `codex_run` 类任务按需注入代理

### 5.3 NO_PROXY

- [ ] 已确认 `NO_PROXY` 至少覆盖 `127.0.0.1`
- [ ] 已确认 `NO_PROXY` 至少覆盖 `localhost`
- [ ] 已确认 `NO_PROXY` 至少覆盖 `::1`
- [ ] 已确认 `NO_PROXY` 至少覆盖 `redis`
- [ ] 已确认内部服务名不会被代理劫持

## 6. Tunnel 检查

- [ ] 已确认 Cloudflare 账号和域名可用
- [ ] 已确认 Tunnel 只转发到本地 API
- [ ] 已确认不会通过 Tunnel 暴露 Redis
- [ ] 已确认不会通过 Tunnel 暴露 worker 内部接口

## 7. 槽位与调度检查

- [ ] 已确认采用 `4 个固定槽位`
- [ ] 已确认 `win-a -> slot-a`
- [ ] 已确认 `win-b -> slot-b`
- [ ] 已确认 `win-c -> slot-c`
- [ ] 已确认 `win-d -> slot-d`
- [ ] 已确认第一版不启用借槽

## 8. 任务模型检查

- [ ] 已确认任务提交为异步模式
- [ ] 已确认客户端不会同步等待长任务完成
- [ ] 已确认每个任务都有 `task_id`
- [ ] 已确认每个任务都有状态查询能力
- [ ] 已确认任务类型采用白名单
- [ ] 已确认不会开放任意命令执行

## 9. 上线顺序检查

- [ ] 已确认先起 Redis
- [ ] 已确认再起 API
- [ ] 已确认先只验证一个槽位
- [ ] 已确认本地链路通过后再接 Tunnel
- [ ] 已确认全部通过后再扩到四槽

## 10. 风险确认

- [ ] 已确认如果 `mac mini` 宕机，四台客户端都会受影响
- [ ] 已确认第一版是单点架构
- [ ] 已确认当前目标是先稳跑通，不是先做高可用
